# Исследование возможностей GRU-архитектур для эмоционального мониторинга по клавиатурному вводу

<img src="imgs/emotions.png" width="30%"/>

> Цель исследования: оценить эффективность применения архитектуры GRU для распознавания эмоционального состояния пользователя по динамическим характеристикам нажатий клавиш.


## Рассматриваемый корпус

[EmoSurv Keystroke dynamics](https://ieee-dataport.org/open-access/emosurv-typing-biometric-keystroke-dynamics-dataset-emotion-labels-created-using)

- **Данные:** нажатиях клавиш клавиатуры с метками эмоций
- **Задача:** классификация
- **Разметка:** эмоции (Neutral, Calm, Happy, Angry, Sad)


## Задачи

**Часть 1:** [`baseline`](1-baseline.ipynb)

1. Предварительная обработка данных, создание дополнительных признаков
2. Анализ данных
3. Обучение и сравнительная оценка GRU-моделей с различной архитектурой

**Часть 2:** [`improvements`](2-improvements.ipynb), [`samplers`](2-samplers.ipynb)

1. Исследование методов борьбы с переобучением и  дисбалансом классов для повышения качества распознавания редких эмоций

**Часть 3:** [`GRU-analysis`](3-GRU-analysis.ipynb)

1. Визуализация и интерпретация полученных результатов

## Ограничения модели

1. Недостаточное количество признаков

Из 11 признаков 7 малоинформативны. Это ограничивает способность модели разделять эмоциональные состояния.

2. Естественное перекрытие эмоциональных кластеров в пространстве признаков

Эмоции, близкие по валентности и возбуждению (например, Angry и Sad), демонстрируют схожие поведенческие паттерны, что подтверждается высоким процентом ошибок между этими классами.

3. Индивидуальные различия в стиле набора текста

Различия (базовая скорость, частота ошибок) создают дополнительный шум, который модель не может компенсировать без персонализированной калибровки.

## Выводы

1. **Эффективность**

GRU-архитектуры с механизмами внимания могут быть пригодны для распознавания эмоций по нажатиям клавиш, однако эффективность модели ограничена качеством и количеством признаков, а также естественной сложностью задачи.

2. **Применимость**

GRU-архитектуры могут использоваться для поставленной задачи не как универсальное решение, а как часть более сложной системы, включающей дополнительные источники признаков и персонализированную калибровку.
